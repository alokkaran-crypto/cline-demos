# Instructions for Task Management with Cline

When beginning a new task using cline, follow these best practices to ensure a clear, trackable, and effective workflow.

---

## 1. Create and Maintain a Task Progress Checklist

- Start every task by making a checklist of all the major steps required.
- Use Markdown format with `- [ ]` for incomplete items and `- [x]` for items you complete as you go.
- Keep the checklist updated throughout your work for maximum clarity.

### Example Checklist Structure

```markdown
- [ ] Analyze requirements
- [ ] Set up necessary files
- [ ] Implement main functionality
- [ ] Handle edge cases
- [ ] Test the implementation
- [ ] Verify results
```

---

## 2. Why Use a Checklist?

- **Roadmap:** Provides a clear breakdown of all work to be done.
- **Progress Tracking:** Lets you and others see at a glance what’s finished.
- **Accountability:** Prevents overlooking or forgetting steps.
- **Transparency:** Users can monitor and, if needed, adjust the checklist.

---

## 3. How To Use Your Checklist

- Include a `task_progress` checklist in your initial planning.
- Update the checklist after finishing each step.
- Review the checklist before marking the task as complete to ensure all items are checked off.

---

## 4. Best Practices

- Make checklist items action-oriented and meaningful—avoid overly granular technical steps.
- If a change in scope occurs, adjust your checklist accordingly.
- For complex tasks, break items down into logical subtasks for clarity.

---

**Keeping your task progress updated and visible ensures smooth collaboration and successful, complete results with cline.**
